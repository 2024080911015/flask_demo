# 产品需求文档 (PRD) - 社交好友推荐系统 Web 端

## 1. 产品概述
* **项目背景**：目前小组已经完成了基于 PyTorch 和用户特征（Embeddings）的推荐算法模型，并封装成了 Flask 接口。现需要开发前端 Web 页面，将抽象的 JSON 数据转化为直观、易用的交互界面。
* **产品目标**：提供一个简洁的查询页面，让用户输入学号（ID）即可快速获取算法推荐的好友列表，并展示相关匹配信息。

## 2. 用户角色
* **普通用户（学生）**：希望通过输入自己的 ID，发现潜在的、兴趣相投的同学。
* **开发者/测试者（小组成员）**：需要查看算法推荐的合理性，浏览所有用户库数据。

## 3. 功能需求梳理

### 核心功能 (P0 优先级 - 必须实现)
* **用户搜索模块**
    * **功能描述**：提供一个输入框，允许用户输入数字格式的 `student_id`。
    * **交互规则**：点击“推荐”按钮或按下回车键触发查询。输入为空或非数字时，需有前端弹窗提示拦截。
    * **对应接口**：无需直接调接口，作为后续请求的参数来源。
* **推荐结果展示模块**
    * **功能描述**：展示当前查询用户的信息，以及 Top-K（目前后端设为 5）的推荐好友列表。
    * **交互规则**：推荐好友以“卡片”形式排版，清晰展示好友的详细信息。在等待后端响应期间，需有明显的“加载中”提示。
    * **对应接口**：`GET /tuijian?id={sid}`

### 扩展功能 (P1 优先级 - 提升体验)
* **用户档案大厅**
    * **功能描述**：提供一个单独的页面或抽屉栏，分页展示系统中所有的用户基础信息。
    * **对应接口**：`GET /users`
* **异常状态反馈**
    * **功能描述**：当输入的 ID 在系统中不存在，或者后端服务未启动时，页面不能崩溃，需要显示友好的错误提示（如：“未找到该用户，请检查 ID 是否正确”）。

### 进阶功能 (P2 优先级 - 小组加分项)
* **推荐关系网络可视化**
    * **功能描述**：推荐系统本质上是计算节点间的连接。除了卡片列表，可以考虑增加一个图表视图，将当前用户置于中心，推荐用户作为周围节点，通过连线来直观展示这种关系网络。

## 4. 界面与交互规范 (UI/UX)
* **设计风格**：现代化、清爽。推荐使用卡片式设计（Card UI）来分割不同用户的信息。
* **响应式设计**：页面需要同时适配电脑端浏览器和手机端屏幕。

## 5. 前后端接口对接映射

| 前端功能模块 | 调用的后端路由 | 预期接收的参数 | 预期处理的返回数据 |
| :--- | :--- | :--- | :--- |
| **获取推荐** | `/tuijian` | `id` (int) | `student_info` (当前信息), `recommend_friends` (数组) |
| **获取总名单** | `/users` | 无 | 对象数组 `[{student_id, student_info}]` |
| **单用户查询** | `/user` | `id` (int) | `student_id`, `student_info` |
